# Play Console — Data safety form answers

The app is **local-first**. For users who never sign in, **no data leaves the device**.
Data is collected only when a user opts into cloud backup/sync (email or Google sign-in).

## Does your app collect or share any of the required user data types?
**Yes** (only for users who choose to create an account / sync).

## Data collected
| Data type | Collected | Shared | Optional? | Purpose |
|---|---|---|---|---|
| Email address | Yes (only if signing in) | No | Optional (only for account) | Account management, App functionality |
| Health & fitness info (your smoking logs / habit data) | Yes (only if syncing) | No | Optional (only if backup enabled) | App functionality (backup & multi-device sync) |
| App activity (in-app settings/preferences) | Yes (only if syncing) | No | Optional | App functionality |

## Data NOT collected / transmitted
- No location, contacts, photos, files, messages, or device identifiers.
- No advertising ID. **No ads.**
- No third-party analytics or crash reporting that transmits personal data
  (diagnostics are kept on-device only).

## Security practices
- **Data is encrypted in transit** (HTTPS/TLS). Yes.
- **Users can request that data be deleted** — Yes. In-app "Clear all data" removes
  local and synced data; account deletion available on request via the contact email.
- Data access is restricted per user (row-level security; a user can only ever read
  their own rows).

## Data sale
- **We do not sell user data.** No.

## Families / children
- Not directed at children. See content rating (tobacco references → Teen+).
